How to Get Run Time User Input in Custom Report?

  • You can give inputs for the report while generating report by using the Macros, You can add the bellow listed macros for runtime user inputs.
  • General Macro format: The Macro's should start with '[Macro_' and Ends with ']'. the Text which is in front of control is followed by macro start text,then Control type like Int,TextBox,Browse,...etc. and then ENDTEXT or ET for the end text which is followed by the control.
  • TextBox: If you need text input, add below macro to load TextBox control. 
    [Macro_UserName_Text]                          -> Load TextBox followed by UserName.
  • Integer Control: If you need number as input, add below macro to load Numeric control. 
    [Macro_Age_Int]                      -> Include Numeric control followed by text 'Age'
    [Macro_Inactive in last_Int_et:days] -> Include Numeric control with End text 'days'
  • DropDown: This macro loads DropDown control. You need to provide dropdown items as semi-colon separated values with the parameter 'Options:'. 
    [Macro_IsLicensed_DropDown_Options:True;False] -> DropDown controle loads 'True' and 'False' from this Macro.
  • DateTime Control: You can add Date,Time,Datetime and DateRange controls by using the following Macros. 
    [Macro_DOB_DateTime]                 -> Include DateTime control followed by 'DOB' and return DateTime.
    [Macro_To_DateRange]                 -> if you need two dates, use this macro along with DateTime macro to get Start and End datetime. 
    [Macro_DOB_Date]                     -> this will return onle date value.
    [Macro_DOB_Time]                     -> this will return onle time value.
  • Target: This control get values from the given target and populate values in dropdown. You can select value from dropdown. 
    [Macro_User Name_Target_ADUserInfo_DisplayName]-> Dropdown Items loads DisplayName from ADUserInfo Objetcs from Database.
  • Browse: This control provide Browse option to select any target from database.if you need to provide Browse option to select multiple Opjects. The last part of the macro should include 'True' 
    [Macro_Name_Browse_ADUserInfo_DisplayName]    -> Provide option to Browse objects from ADUserInfo in Database.
    [Macro_Name_Browse_MailBoxInfo_DisplayName]   ->  Provide option to Browse objects from MailBoxInfo in Database.
  • User Picker: This control provide Browse option to select users from the target ADUserInfo in database. You need to provide required column to select in the final part of the macro. if You Need to Provide Browse option to select multiple users. The last part of the macro 'True' decides whether this is single or multi valued. default is single. 
    [Macro_User_UserPicker_UserPrincipalName]     -> Provide option to browse user from ADUserInfo and return single 'UserPrincipalName'
    [Macro_User_UserPicker_UserPrincipalName_True]-> Provide option to browse users from ADUserInfo and return 'UserPrincipalName' collection as Powershell Array.
  • External Resourse: Use below macros to get CSMO binaries 
    [Macro_Include_CSMOAssembly]        -> Include Csom Assembly's
    [Macro_Include_SPOAdminUrl]         -> Include Sharepoint Tanent Admin Url

    How to Configure Advance Macros in Custom Report?

  • Powershell Modules: Use below macros to load required PowerShell modules. 
    [Macro_LoadPSModule_ExchangeOnline] -> Office 365 Exchange Module
    [Macro_LoadPSModule_MSOnline]       -> Azure AD Module
    [Macro_LoadPSModule_SPOnline]       -> Sharepoint Online Module
  • Credential: Use below macros to load credentials in different ways. 
    [Macro_Include_SecureCredential]    -> Include UserName,Password,Password SecureString
    [Macro_Include_PSCredential]        -> Include Powershell Credential
